# 五、对嵌入式最重要的 C 语言知识点

你可以把下面这些当作“考试大纲”。

## 1. 指针

必须掌握：

```c
int *p;
*p;
&p;
p++;
p[i];
```

还要理解：

```c
uint8_t *buf;
const uint8_t *p;
uint8_t *const p;
void *ptr;
```

嵌入式里缓冲区、驱动接口、DMA、串口接收都会用。

---

## 2. 数组

重点理解：

```c
uint8_t rx_buf[128];
uint16_t adc_value[16];
char cmd[32];
```

必须知道数组名在多数情况下会退化为指针。

---

## 3. 字符串

串口命令解析会经常用：

```c
char cmd[32];

if (strcmp(cmd, "led on") == 0) {
    LED_On();
}
```

需要掌握：

```c
strlen()
strcmp()
strncmp()
strcpy()
strncpy()
strstr()
sscanf()
```

但要注意，嵌入式里字符串函数容易越界，必须控制缓冲区长度。

---

## 4. 结构体

这是嵌入式项目组织代码的核心。

比如：

```c
typedef struct {
    float angle;
    float gyro;
    float acc;
} IMU_Data_t;
```

或者：

```c
typedef struct {
    float kp;
    float ki;
    float kd;
    float target;
    float output;
} PID_t;
```

以后你写传感器驱动、电机控制、协议解析，都会大量用结构体。

---

## 5. 位运算

嵌入式必须熟练：

```c
#define BIT0    (1 << 0)
#define BIT1    (1 << 1)

state |= BIT0;     // 置位
state &= ~BIT0;    // 清零
state ^= BIT0;     // 翻转
if (state & BIT0)  // 判断
```

寄存器配置、状态标志、通信协议都离不开位操作。

---

## 6. 宏定义

必须会：

```c
#define LED_ON()   HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_SET)
#define LED_OFF()  HAL_GPIO_WritePin(GPIOA, GPIO_PIN_5, GPIO_PIN_RESET)
```

还要会条件编译：

```c
#ifdef DEBUG
printf("debug info\r\n");
#endif
```

---

## 7. `static`、`extern`、`volatile`

这三个特别重要。

### `static`

用于限制变量或函数作用域：

```c
static uint8_t rx_index = 0;
```

### `extern`

用于跨文件引用变量：

```c
extern uint8_t rx_buf[128];
```

### `volatile`

用于中断、寄存器、硬件状态变量：

```c
volatile uint8_t uart_rx_flag;
```

面试非常爱问 `volatile`。

---

## 8. 函数指针

先不用深挖，但要能看懂这种形式：

```c
void (*callback)(void);
```

后面你看 HAL 库、RTOS、驱动框架时会遇到。

# 七、你不需要做到什么程度？

不要追求这些：

1. 不需要背完整语法。
    
2. 不需要刷大量 C 语言竞赛题。
    
3. 不需要深挖复杂数据结构。
    
4. 不需要熟练 `malloc/free`。
    
5. 不需要一开始看懂 Linux 内核代码。
    
6. 不需要把《C Primer Plus》所有习题做完。
    

你真正需要的是：

> 能写清晰的 C 代码，能用结构体组织模块，能用指针处理缓冲区，能用位运算操作标志位，能用宏和多文件工程写驱动。
